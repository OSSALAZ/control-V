Levantamiento de Requerimientos - Control Vehicular
	
Objetivos del Proyecto
4.1 Descripción general del objetivo del Cliente
Implementar un sistema piloto de control y verificación de vehículos autorizados para circular en avenidas de uso restringido dentro del municipio, mediante la acreditación de unidades registradas a domicilios locales y el uso de etiquetas RFID, complementado con validación visual y herramientas tecnológicas básicas. 
Este piloto permitirá al municipio iniciar la reducción del congestionamiento vehicular, evaluando la viabilidad operativa y tecnológica del modelo que, a largo plazo, evolucionará hacia un sistema integral de cobro por acceso y gestión de movilidad urbana.
4.3 Alcance Esperado:
El proyecto piloto abarcará la implementación de un sistema básico pero funcional para la acreditación, registro y verificación de vehículos pertenecientes a residentes del municipio, con el objetivo de habilitar el uso exclusivo de determinadas avenidas. El alcance contempla:
Diseño e implementación de una base de datos centralizada

Registro de vehículos autorizados.
Asociación de cada unidad con su propietario y domicilio en el municipio.
Integración del número de serie único del tag RFID asignado a cada vehículo.
Entrega y colocación controlada de etiquetas RFID y calcomanías visuales

Producción y distribución de tags de bajo costo.
Adhesión del chip y calcomanía en puntos de acreditación oficiales.
Captura de información del vehículo y validación de residencia y predial.

Instalación de puntos de verificación en campo

Lectura de tags RFID en avenidas con circulación restringida.
Alerta visual o señalización para cuerpos de tránsito cuando un vehículo no autorizado acceda.
Verificación complementaria mediante inspección visual de calcomanía.


Integración mínima con sistemas del municipio


Consulta para validación de pago de predial.
Acceso a registros necesarios para confirmar la acreditación del propietario.

Operación del piloto en un periodo estimado inicial de un mes

Capacitación básica al personal municipal involucrado.
Pruebas de funcionamiento del sistema de registro y verificación.
Ajustes menores para optimizar el flujo operativo.

Este alcance permitirá validar la viabilidad del modelo antes de escalarlo hacia un sistema completo de restricción vehicular y cobro por acceso, previsto en la fase del proyecto macro.
5. Requerimientos
5.1 Requerimientos Funcionales
Registro y acreditación de vehículos
El sistema debe permitir registrar vehículos cuyos propietarios acrediten domicilio en el municipio.

Debe capturar datos del propietario (nombre, identificación, domicilio) y del vehículo (placas, modelo, color, serie).

Debe almacenar el número de serie único del tag RFID asignado a cada vehículo.
Validación de residencia y predial
El sistema debe permitir consultar, de forma manual o mediante interfaz, la base de datos municipal para verificar que el predial esté pagado.

Debe almacenar la evidencia de acreditación para auditoría futura.
Asignación y control de RFID
El sistema debe registrar la asignación del RFID a un vehículo específico.
Debe permitir la lectura del chip RFID en los puntos de acreditación y verificación.

Debe registrar la fecha y lugar de instalación del tag.
Emisión y control de calcomanías
El sistema debe relacionar cada calcomanía holográfica/visual con el RFID y el vehículo asignado.

Debe registrar la entrega e instalación de calcomanías en los módulos municipales.
Verificación en campo
El sistema debe permitir leer RFID en accesos restringidos mediante antenas/lectores.

Debe determinar automáticamente si un vehículo está o no autorizado para circular por una vía restringida.

Debe emitir una alerta visual o señal para el personal de tránsito cuando un vehículo no autorizado pase.
Consulta para autoridades de tránsito
El sistema debe permitir que una patrulla consulte el estatus de un vehículo mediante:


RFID

Placas

Calcomanía visual

Debe mostrar si es un vehículo autorizado o no para esa zona.
Generación de reportes
El sistema debe generar reportes básicos de:

Vehículos acreditados.

RFID instalados.

Intentos de acceso no autorizado (si aplica tecnología para registro).

Actividad por puntos de acreditación.

5.2 Requerimientos No Funcionales
Disponibilidad
El sistema debe estar disponible durante la operación del piloto (mínimo 8 a 12 horas diarias).
Fiabilidad
La lectura RFID debe ser consistente en un porcentaje aceptable (ej. 90% o más para el piloto).
Escalabilidad
La solución debe diseñarse para poder crecer hacia el proyecto macro sin rehacer todo desde cero.

Seguridad
Los datos de propietarios y vehículos deben almacenarse de forma segura.
Debe incluir controles básicos de acceso para evitar modificaciones no autorizadas.
Usabilidad
La interfaz de registro debe ser simple y usable por personal no técnico.


El proceso de acreditación debe ser rápido (idealmente menos de 5 minutos por vehículo).

Compatibilidad
El sistema debe ser capaz de integrarse con la base de datos municipal de predial mediante API, consultas web o mecanismos acordados.

Mantenimiento
El sistema debe permitir ajustes menores sin interrumpir la operación del piloto.

6. Supuestos
El municipio proporcionará acceso a la información necesaria para validar pago de predial.

El municipio facilitará los puntos físicos para la acreditación de vehículos.

Los tags RFID estarán disponibles en tiempo y forma (incluyendo importación si aplica).

El personal municipal colaborará con la instalación de RFID y calcomanías.

El proyecto piloto no requiere todavía el sistema de cobro por acceso ni el monedero electrónico.

El municipio aceptará un nivel tecnológico básico para el piloto (lectores RFID + verificación visual).

La infraestructura de energía y red en puntos de verificación será provista por el municipio o ya existe.

7. Restricciones
El presupuesto del piloto debe mantenerse bajo y no exceder costos básicos de materiales y desarrollo.

El tiempo para implementar la solución no debe superar aproximadamente 1 mes.

La solución no incluirá infraestructura compleja como plumas, sensores avanzados, teleféricos ni transporte público gratuito (eso pertenece al proyecto macro).

La verificación mediante IA/cámaras no es obligatoria para el piloto y se incluirá solo si el tiempo y presupuesto lo permiten.

El alcance está limitado geográficamente a ciertas avenidas del municipio.

La importación o tiempo de entrega de los tags RFID puede limitar el inicio del piloto.
